# CLI Implementer (uv-cli-implementer)

This skill designs and implements small, focused, agent-friendly CLI tools using `uv` (PEP 723 inline dependencies) and `Typer`.

## Triggers

The skill is invoked via natural language requests instructing the agent to build a command-line tool. Examples include:

- **"Implement a CLI tool for [task]"**
- **"Create a uv script that does [task]"**
- **"Build an agent-friendly CLI to [action]"**

### Workflow
1. **Clarify Scope:** The agent ensures the requested tool adheres to the Unix philosophy (does exactly one thing well).
2. **Implementation Plan:** The agent produces a plan detailing subfunctions, arguments, and options.
3. **Approval:** The user reviews and approves the plan.
4. **Generation:** The agent writes the `uv`-compatible Python script.

## Standard Output

The skill produces a single-file Python script, typically located at:
- `scripts/<tool-name>.py` (if skill-specific)
- `<workspace>/.agent/tools/<tool-name>.py` (if globally reusable)

### Invocation Pattern

The generated script is designed to be executed directly via `uv` with zero external setup:

```bash
uv run scripts/<tool-name>.py <arg> [--option value]
```

### Agent-Friendly Design
Tools generated by this skill follow strict rules to be usable by other LLM agents:
- **stdout:** Emits structured output (preferably JSON) on success.
- **stderr:** Emits descriptive error messages on failure to aid in agent self-correction.
- **Exit Codes:** Exits with `0` on success and `1` (or other non-zero codes) on failure.
- **Non-Interactive:** Never uses interactive prompts (e.g., `input()`), ensuring seamless automated execution.
- **Self-Documenting:** Fully documented via the `--help` flag for dynamic discovery.

## Suggested Subcommands

To make the workflow more explicit, the following structured commands could be implemented in the future:

- `/cli scaffold <tool-name> "<description>"`
  Bypasses the planning phase and immediately scaffolds the boilerplate `Typer` script with the specified name and description.
- `/cli add-dep <script-path> <package>`
  Parses the target script's PEP 723 header and adds a new dependency, updating the inline requirements automatically.
- `/cli lint <script-path>`
  Runs a suite of checks on the generated script to ensure it meets the "agent-friendly" criteria (no `input()`, correct exit codes, JSON output).
