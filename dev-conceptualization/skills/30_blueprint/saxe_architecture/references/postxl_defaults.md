# PostXL Default Architecture

Every PostXL project ships with this architecture out of the box. Do not
reinvent these — document them as the baseline and extend only where needed.

## Default App Structure

```
project/
├── backend/
│   ├── apps/api/src/           # Single NestJS + Fastify API app
│   │   ├── main.ts             # Bootstrap: Fastify adapter, plugins, tRPC, Swagger
│   │   ├── api.module.ts       # Root module (imports all libs)
│   │   ├── api.config.ts       # Zod-validated config (DB, auth, S3, upload limits)
│   │   └── api.controller.ts   # Health check endpoint
│   │
│   └── libs/                   # One NestJS module per concern
│       ├── database/           # Prisma client wrapper (DatabaseService)
│       ├── repositories/       # Dual repository: DatabaseRepository + InMemoryRepository
│       ├── authentication/     # Keycloak/OIDC: JWT validation, JWKS, session cookies
│       ├── actions/            # Action dispatcher: audit trail + authorization
│       ├── router-trpc/        # tRPC server, plugin registration, context creation
│       ├── update/             # Write layer: per-entity UpdateService (create/update/delete)
│       ├── view/               # Read layer: per-entity ViewService (get/getAll/getFiltered)
│       ├── import/             # Excel/data import with preview + validation
│       ├── upload/             # File upload (multipart → S3)
│       ├── s3/                 # AWS S3 client
│       ├── excel-io/           # Excel preview/export
│       ├── ai/                 # Claude AI integration (agent service, tools, cache)
│       ├── seed/               # Database seeding on startup
│       ├── e2e/                # E2E test utilities
│       ├── restApi/            # REST/Swagger endpoint fallback
│       └── data-management/    # Admin data operations (conditional)
│
├── frontend/
│   └── src/
│       ├── main.tsx            # React 19 entry point
│       ├── routes/             # TanStack Router (file-based, generated routeTree)
│       ├── pages/              # Route page components
│       ├── components/         # Feature components
│       ├── ui/                 # @postxl/ui-components
│       ├── hooks/              # React hooks (data fetching, auth)
│       ├── context-providers/  # Auth, theme, tRPC providers
│       ├── actions/            # tRPC mutation wrappers
│       └── stories/            # Storybook stories
│
└── docker-compose.yml          # API + PostgreSQL + Keycloak
```

## Default Data Flow

```
Frontend (React 19)
  │
  │  tRPC client (HTTP POST, superjson, batched)
  ▼
Fastify/tRPC Server
  │
  │  authMiddleware → JWT from HttpOnly cookie → JWKS validation
  ▼
tRPC Router (per-entity routes)
  │
  ├─ Queries → ViewService → Repository → Prisma → PostgreSQL
  │
  └─ Mutations → DispatcherService
                    │
                    ├─ Create ActionExecution (audit log)
                    ├─ AuthorizationService (RBAC check)
                    └─ UpdateService → per-entity service
                         │
                         ├─ Validate payload (Zod)
                         ├─ Repository.create/update/delete → Prisma → PostgreSQL
                         └─ Push ActionOperation (audit detail)
```

## Default tRPC Route Pattern

Every entity model gets these standard routes (generated by PostXL):

```typescript
entity.get(id)            // query: ViewService.get()
entity.getAll()           // query: ViewService.getMap()
entity.getFiltered(opts)  // query: ViewService.getFiltered()
entity.create(payload)    // mutation: Dispatcher → UpdateService
entity.update(id, data)   // mutation: Dispatcher → UpdateService
entity.delete(id)         // mutation: Dispatcher → UpdateService
```

Plus standard cross-cutting routes:
- `viewer` — current authenticated user info
- `ai` — AI agent interactions
- `auditLogs` — action history
- `excelIo` — import/export operations
- `actions`, `actionOperations` — audit trail queries

## Default Auth Flow

```
Browser → Keycloak login page → redirect with auth code
  → Backend exchanges code for JWT
  → JWT stored in HttpOnly cookie (secure, domain-scoped)
  → Every tRPC request: cookie → JWKS validation → ctx.viewer
  → First login auto-creates User entity via dispatcher
```

## Default NestJS Module Pattern

Each lib follows a consistent pattern:

```
libs/<module>/
├── src/
│   ├── <module>.module.ts       # NestJS @Module declaration
│   ├── <module>.service.ts      # Business logic
│   ├── <module>.repository.ts   # Data access (optional, for entity modules)
│   └── index.ts                 # Public exports
```

Modules are composed via `api.module.ts` which imports all libs.
