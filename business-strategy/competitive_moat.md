# Competitive Moat

## Five Defensibility Layers

### 1. Proprietary Full-Stack Framework (Technical IP)
- Proprietary code generators with unique 3-way file sync (generated vs lock file vs disk)
- Battle-tested in production: 10 apps built by data2impact, including for regulated industries (pharma, banking, biotech)
- Generator approach is unique and hard to replicate
- Full-stack TypeScript: NestJS + React + PostgreSQL + Keycloak, schema-driven

### 2. Generalized Skill Execution Model (Process IP)
- No competitor has structured artifact-based pipelines with configurable phases and approval gates
- The pattern — skill definitions, artifact schemas, checkpoint gates, feedback loops, quality scoring — is proven in the software vertical and generalizes to any structured knowledge-work process
- Each new vertical (consulting, sales, due diligence) adds skill templates that are hard to replicate, while the platform infrastructure stays shared
- This is not a "future vision" — it is the core architecture, with software as the first instance

### 3. Enterprise IT Integration
- Deep customization hooks that consumer-grade AI builders (Bubble, bolt.new, lovable) and general AI assistants can't match:
  - Auth (SSO/SAML/OIDC via Keycloak)
  - Corporate design system
  - Architecture (backend stack is swappable)
  - Deployment (any infrastructure)
  - AI provider (any LLM, including self-hosted)
  - Skill templates (custom pipeline definitions per vertical/use case)
- IT approves the platform once — business units spin up skill pipelines (or open sessions) for their specific needs

### 4. No Vendor Lock-In
- Generated apps are real, standard code (NestJS + React + PostgreSQL)
- Apps run independently of PAXL — can be cloned, customized, deployed anywhere
- Git-backed projects: IT can take over using standard tools
- This is the opposite of Bubble/Mendix lock-in and a key enterprise selling point
